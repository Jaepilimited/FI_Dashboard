# 지역별/상품별 P&L 테이블 — 기간별 Top10 재선정 + 기타 컬럼, 지역별 기본 드릴레벨

## 배경

`renderCategoryPl()` (templates/dashboard_v2.html)은 조직별/지역별/상품별/판매유형별 탭에서 공통으로 쓰는 P&L 테이블 렌더러다. 지역별·상품별 탭은 `cplDims()`가 전체기간 매출 합계 기준으로 고정된 top10 국가/상품만 컬럼으로 보여주고, 나머지(11위 이하)는 화면에서 사라진다. 그 결과:

- 화면에 보이는 top10 컬럼의 합이 "합계" 컬럼과 일치하지 않아 검증이 안 됨
- 기간(월/분기/반기)을 펼쳐도 국가/상품 세트와 순서가 항상 동일해서, 특정 기간에 급성장한 국가가 다른 기간에도 안 보일 수 있음

또한 사이드바에서 "지역별" 탭 진입 시 기본 드릴레벨이 "권역"(가장 상위 레벨)이라 "국가"까지 매번 수동으로 드릴다운해야 한다.

## 범위

- 대상: `region`, `product` 카테고리만 (org는 `renderOrgPl()`에서 이미 전체 부서를 다 보여주므로 top10 캡이 없어 대상 아님)
- `renderCategoryPl()` 내부 로직 변경 + `state.viewLevel` 초기값 변경만. 백엔드(API), `cplDims()` 자체, CSV export 엔드포인트는 변경하지 않음

## 변경 1 — 기간별 Top10 재선정 + 기타 컬럼

### 현재 동작
`dims = cplDims()`가 반환하는, 전체기간 매출 합계 기준 정적 top10 리스트(`secDims`)를 모든 기간(월/분기/반기)에서 동일하게 사용. 컬럼 헤더/데이터 셀 모두 이 고정 리스트를 순회.

### 변경 후 동작
- 각 기간이 "펼쳐질" 때(`isPeriodOpen`), **그 기간 자체의 매출 합계 기준**으로 전체 노드(국가/상품) 중 top10을 다시 산정한다. 컬럼 "개수"는 항상 `min(10, 전체 노드 수)`로 고정되지만, 어떤 노드가 그 10개에 들어가는지와 순서는 기간마다 달라질 수 있다.
- "기타" 컬럼(전체 노드 수 > 10일 때만 표시) = 그 기간의 "합계"(`totM`, 변경 없음 — 항상 전체 노드 합) − 그 기간에 선정된 top10 값의 합. 따라서 `top10 합 + 기타 = 합계`가 항상 성립.
- 전월(전기간) 대비 ▲▼ 비교는 같은 노드 이름 기준으로 이전 기간 값을 조회한다 — 이전 기간에 그 노드가 top10에 없었어도(즉 그 기간엔 화면에 없었어도) 값 자체는 조회 가능하므로 비교는 그대로 성립한다.
- "기타"의 ▲▼ 비교는 "기타" 자체를 하나의 가상 노드로 보고, 현재 기간 기타값과 이전 기간 기타값(그 기간 나름의 top10 기준으로 산출한 값)을 비교한다.
- %행(비중)은 기존과 동일한 패턴 유지: 각 컬럼(기타 포함) 값 ÷ 그 컬럼의 매출값 × 100. 기타의 매출값도 동일하게 "그 기간 매출 합계 − top10 매출 합"으로 산출.
- 컬럼 개수가 기간마다 고정(10 + 기타 있으면 +1 + 계)이므로 colspan 등 테이블 구조 코드는 대부분 그대로 재사용 가능.

### 계산 예시
2026-01에 매출 top10에 미국·중국·... 10개국이 들어가고, 2026-02엔 그중 일부가 빠지고 다른 나라가 새로 들어갈 수 있음. 각 기간은 독립적으로 "그 기간의 매출 1~10위"를 기준으로 컬럼을 채운다.

### 구현 메모 (참고용, 계획 단계에서 확정)
- 섹션(전체/SK/UM) 별로 랭킹이 달라지므로 `CPL_SEC_DEFS.forEach` 루프 안에서 섹션별로 재계산
- 이름별 `cplMetrics(sec, name)` 결과를 섹션 스코프에서 메모이즈해서 중복 계산 방지
- 기존 `secDims`/`dcols`/`topM`(정적, 인덱스 기반) 로직은 제거하고, 기간별 `periodTopNames[prd.key]` (이름 배열, 기간마다 재계산) + 이름 기반 값 조회 함수로 대체
- 우클릭 CSV 다운로드(`data-plitem`/`data-pldim`/`data-pldimval`)는 top10 컬럼엔 기존과 동일하게 유지(노드 이름으로), "기타" 컬럼엔 부여하지 않음(단일 값으로 필터링 불가능한 잔차 합계이므로) — 클릭해도 아무 동작 없는 일반 셀

## 변경 2 — 지역별 기본 드릴레벨

`state.viewLevel` 초기값을 다음과 같이 변경:

```js
viewLevel: { org: 0, region: 2, product: 0 },
```

`region`의 `path`가 `['권역','대륙','국가','거래처']`이므로 index 2 = "국가". 세션 중 사용자가 다른 레벨로 드릴하면 그 값이 유지되며(기존 동작과 동일), 이 변경은 오직 최초 상태값 한 줄만 바뀐다. `switchCategory()`나 다른 곳에서 `viewLevel`을 별도로 리셋하는 로직이 없음을 확인함 — 추가 변경 불필요.

## 영향받지 않는 부분

- `cplDims()` 함수 자체(가드용 호출은 유지하되 반환값의 top10 순서/내용은 이 변경 이후 지역별/상품별 렌더링에는 더 이상 쓰이지 않음)
- `renderOrgPl()` (조직별) — 완전히 별도 로직, 변경 없음
- 백엔드 API(`/api/pl`, `/api/pl-source-export`) — 변경 없음, 프론트엔드에서 이미 받은 `state.cplData.byNode` 데이터를 프론트에서 재정렬/집계만 함
- 사이드바 트리 레일(`renderTreeRail`)의 국가 정렬 로직 — 별도 컴포넌트, 변경 없음

## 테스트 관점
- 지역별/상품별 탭에서 여러 기간을 동시에 펼쳤을 때 각 기간별로 top10 국가/상품 세트와 순서가 그 기간 매출 기준으로 다른지 확인
- 각 기간에서 "top10 합 + 기타 = 합계" 컬럼 값이 실제로 일치하는지 확인 (매출액 행 기준으로 손으로 검산)
- 전체 노드 수가 10개 이하인 필터 조합(예: 특정 월+특정 브랜드로 좁혔을 때)에서 기타 컬럼이 사라지는지 확인
- 기타 컬럼의 전월 대비 ▲▼가 이상치 없이 표시되는지 (이전 기간 기타값 재계산 검증)
- 사이드바에서 지역별 최초 진입 시 국가별 화면이 바로 뜨는지, 이후 거래처별로 드릴다운했다가 다른 탭 갔다 오면 거래처별이 유지되는지
